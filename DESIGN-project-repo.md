# 项目储存库 — 设计说明

## 1. 目标

在"首页点击模块"与"进入工作台"之间插入一个**项目储存库**视图，让面试官可以：
- 新建项目并命名
- 查看之前手动保存的项目列表
- 选择一个项目继续编辑

纯前端流程，存储介质为 `localStorage`。**与之前的自动草稿不同——这是用户主动触发的"保存项目"行为，语义清晰。**

---

## 2. 数据结构

```ts
// src/lib/projectStore.ts

interface ProjectMeta {
  id: string;            // 唯一 ID，时间戳 + 随机
  name: string;          // 用户为项目命名，如"《长安十二时辰》改编"
  module: ModuleKey;     // original | rewrite | adapt | evaluate
  createdAt: number;     // Date.now()
  updatedAt: number;     // 最后一次保存时间
  step: number;          // 当前步骤索引
  completed: boolean[];  // 各步骤完成状态
}

interface ProjectSnapshot {
  meta: ProjectMeta;
  workbenchData: Record<string, unknown>; // 同现有 workbenchData
}
```

**存储键名：** `frame_projects` → `ProjectSnapshot[]`

---

## 3. 页面流转

```
首页 → 点击模块卡片 → 项目储存库页面 → 工作台
                              ↑              │
                              └── 保存后返回 ──┘
```

### 3.1 状态机

在 `App.tsx` 中增加一个 view：

```ts
type View = "home" | "project-repo" | "workbench";
```

- `"home"` → 首页
- `"project-repo"` → 项目储存库（中间页）
- `"workbench"` → 工作台

### 3.2 路由参数

```ts
// App.tsx 新增状态
const [pendingModule, setPendingModule] = useState<ModuleKey | null>(null);
const [editingProjectId, setEditingProjectId] = useState<string | null>(null);
```

- 首页点击模块 → `setPendingModule(module)` + `setView("project-repo")`
- 项目储存库中"新建" → `setView("workbench")`，工作台用 `FRESH[pendingModule]` 初始化
- 项目储存库中"继续编辑" → `setEditingProjectId(id)` + `setView("workbench")`，工作台加载存档

## 4. 项目储存库页面设计

### 4.1 布局

```
┌─────────────────────────────────────────────────┐
│  [← 返回首页]          项目储存库                │
│                                                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │ 新建项目  │  │ 剧本原创  │  │ 网文改编  │  ...  │
│  │   ＋     │  │  2 个项目 │  │  1 个项目 │       │
│  │          │  │          │  │          │       │
│  └──────────┘  └──────────┘  └──────────┘       │
│                                                  │
│  ── 剧本原创 ──────────────────────────────      │
│                                                  │
│  ┌─────────────────────────────────────────┐     │
│  │ 🎬 《剑来》改编        6月28日 · 第2步   │     │
│  │ ───────────────────────────────────── │     │
│  │ 🎬 未命名项目           6月27日 · 第1步   │     │
│  └─────────────────────────────────────────┘     │
└─────────────────────────────────────────────────┘
```

### 4.2 模块筛选标签

顶部一排标签按钮，按模块类型筛选项目列表：

| 标签 | 含义 |
|------|------|
| 全部 | 显示所有模块的项目 |
| 剧本原创 | 只显示 original |
| 剧本改写 | 只显示 rewrite |
| 网文改编 | 只显示 adapt |
| 剧本评估 | 只显示 evaluate |

当前选中标签高亮（accent 色），其余为灰色。

### 4.3 新建项目卡片

每列最前面是一个"新建项目"卡片，点击后：

1. **弹出命名对话框**（轻量级 modal）
   - 标题："新建项目"
   - 输入框：项目名称（预填 `未命名项目`，可编辑）
   - 两个按钮：取消 / 创建
2. 确认后 → 创建新 ProjectSnapshot → 写入 localStorage → 进入工作台
3. 如果用户直接回车或点击创建 → 立即进入

### 4.4 项目列表卡片

每个已保存项目的展示：

- **图标 + 项目名**：左侧模块图标（四角星✧等），右侧项目名称
- **元信息行**：模块标签 + 进度（如"第2步 · 故事梗概"）+ 最后保存时间
- **hover 操作按钮**：继续编辑 / 删除（确认对话框）
- **点击卡片**：进入继续编辑

### 4.5 空状态

如果某个模块下没有任何项目，显示引导文案：

> 还没有项目。点击上方"新建项目"开始创作。

## 5. 工作台集成

### 5.1 初始化逻辑

工作台需要接收两种模式：

```ts
interface WorkbenchProps {
  onExit: () => void;
  // 新建模式
  initialModule: ModuleKey;
  // 加载模式（可选）
  projectId?: string;
  projectData?: ProjectSnapshot;
}
```

- `projectId` 存在 → 从 snapshot 恢复 module/step/completed/workbenchData
- `projectId` 不存在 → 使用 FRESH 新建

### 5.2 保存按钮

侧边栏底部新增一个"保存项目"按钮：

```
┌──────────┐
│  保存项目  │  ← 点击后立即将当前状态写入 localStorage
└──────────┘
```

保存逻辑：
1. 对比 `projectId`：有 → 更新已有项目；无 → 创建新项目
2. 如果是新项目 → 弹出命名对话框（同新建流程）
3. 保存成功后 → 按钮短暂变为 "已保存 ✓"（1.5s 后恢复）
4. 如果已有项目名 → 直接覆盖保存，无弹窗

### 5.3 顶栏显示项目名

工作台顶栏 "工作台 / 剧本原创" 改为 "项目名 / 剧本原创"：

> 《剑来》改编 / 剧本原创

## 6. 交互细节

### 6.1 命名对话框

```
┌────────────────────────────┐
│        新建项目             │
│                            │
│  ┌──────────────────────┐  │
│  │ 为这个项目取个名字…   │  │
│  └──────────────────────┘  │
│                            │
│       [取消]   [创建]      │
└────────────────────────────┘
```

- 背景半透明遮罩
- 输入框自动 focus
- Enter 键提交，Esc 键取消
- 点击遮罩外区域取消

### 6.2 删除确认

```
┌────────────────────────────┐
│       确认删除              │
│                            │
│  删除后无法恢复。           │
│  确定要删除"《剑来》改编"？  │
│                            │
│       [取消]   [删除]      │
└────────────────────────────┘
```

### 6.3 保存反馈

工作台侧边栏保存按钮的微交互：
- 点击 → "保存中…"（如果写入快则跳过）
- 成功 → "已保存 ✓"（1.5s 后恢复"保存项目"）
- 失败 → "保存失败"（2s 后恢复，红色文字）

### 6.4 返回路径

- 工作台 → 返回 → 项目储存库（而非首页）
- 项目储存库 → 返回首页 → Home
- 浏览器直接访问 → 始终从首页开始

## 8. 关键设计决策

1. **localStorage 而非 sessionStorage**：面试官关闭浏览器后下次打开仍能看到项目，体现"持久化"概念
2. **手动保存而非自动保存**：与之前移除的 auto-save 不同，手动保存是用户的有意识行为，语义明确，不会造成"打开就是旧数据"的困扰
3. **项目储存库作为中间页**：而非首页直接显示项目列表——保持首页的 Hero 展示效果（面试官第一印象），项目列表作为二级页面
4. **模块筛选而非混合列表**：4 个模块的项目分开管理，逻辑清晰
5. **命名对话框统一复用**：新建项目、首次保存、重命名共用同一个 NameDialog 组件
